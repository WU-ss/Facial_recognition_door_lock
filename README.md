# Facial_recognition_door_lock

基于 **ESP32-CAM + STM32** 的人脸识别多功能家居门禁系统。项目通过 ESP32-CAM 完成图像采集、人脸检测、人脸录入与识别，并在识别通过后控制继电器驱动电控锁开锁；同时使用 STM32 处理红外检测、光照检测、LCD1602 显示、门铃、LED 补光和语音/报警等辅助功能。

本项目适用于智能门禁、DIY 智能门锁、嵌入式视觉识别、智能家居课程设计和实验室门禁原型开发。

> 本项目仅用于学习、实验和原型验证，不建议直接用于真实安防场景。

---

## 项目简介

`Facial_recognition_door_lock` 是一套基于人脸识别的智能门禁系统。系统以 ESP32-CAM 为图像采集与识别核心，通过摄像头实时采集画面，并利用轻量级人脸检测模型对人脸进行检测、录入与识别；当识别结果与已登记用户匹配时，ESP32-CAM 输出控制信号，驱动继电器控制电子锁打开。

系统还集成了 STM32 辅助控制部分，用于实现人体红外检测、光照检测、智能补光、门铃、LCD 状态显示、语音提示和节能控制等功能，使门禁系统不仅能完成“刷脸开锁”，还具备较完整的智能家居交互能力。

---

## 核心功能

- ESP32-CAM 摄像头图像采集；
- 本地 Web 页面视频流显示；
- 人脸检测、人脸录入与人脸识别；
- 支持添加、删除已录入人脸；
- 识别成功后继电器控制电控锁解锁；
- STM32 辅助控制门铃、红外感应、光照检测和状态显示；
- HC-SR505 红外模块检测人体靠近；
- BH1750 / 光照检测模块判断弱光环境；
- 弱光且有人时自动开启高亮 LED 补光；
- LCD1602 显示 `Waiting` / `Welcome` 等状态信息；
- ISD1820 语音模块进行开锁提示；
- 蜂鸣器实现门铃或报警提示；
- 长时间无操作自动关闭部分功能，降低功耗。

## 系统架构

```text
用户靠近门禁
    ↓
红外传感器检测人体
    ↓
STM32 更新 LCD 状态 / 判断是否需要补光
    ↓
ESP32-CAM 采集图像
    ↓
MTMN / CNN 人脸检测与识别
    ↓
识别通过？
    ├── 是：GPIO 输出高电平 → 继电器动作 → 电控锁解锁 → 语音提示
    └── 否：保持上锁 / 提示未识别
```

辅助功能流程：

```text
红外检测 → 判断是否有人 → LCD 显示 Waiting / Welcome
光照检测 → 判断是否弱光 → 自动开启 LED 补光
按键检测 → 蜂鸣器响铃 → 实现门铃功能
长时间无操作 → 自动关闭部分功能 → 节能
```

## 软件设计

### 1. ESP32-CAM 人脸识别程序

ESP32-CAM 端使用 Arduino IDE 开发，主要完成：

- 摄像头初始化；
- WiFi 连接；
- Web 服务器创建；
- 浏览器端视频流传输；
- 人脸检测；
- 人脸录入；
- 人脸识别；
- 人脸特征保存与删除；
- 继电器 GPIO 控制；
- 开锁后定时自动复位。

主要文件：

```text
face_test.ino
camera_index.h
camera_pins.h
```

ESP32-CAM 程序中使用的典型参数包括：

```c
#define ENROLL_CONFIRM_TIMES 5
#define FACE_ID_SAVE_NUMBER 7
#define CAMERA_MODEL_AI_THINKER
#define relay_pin 2
```

其中 `relay_pin` 用于控制继电器，识别成功后输出电平信号触发开锁。

### 2. 人脸检测与识别

项目使用轻量级人脸检测模型完成嵌入式端人脸检测。检测流程大致为：

```text
获取摄像头图像
    ↓
图像格式转换
    ↓
人脸检测
    ↓
人脸对齐
    ↓
提取人脸特征
    ↓
与已登记人脸特征比对
    ↓
识别成功后开锁
```

系统支持以下操作：

- `STREAM CAMERA`：开启视频流；
- `DETECT FACES`：检测人脸；
- `ADD USER`：录入用户人脸；
- `ACCESS CONTROL`：进入门禁识别模式；
- `DELETE ALL`：删除已录入人脸。

### 3. STM32 辅助控制程序

STM32 端主要负责门禁系统的外围辅助功能，包括：

- 按键检测；
- 门铃控制；
- 红外人体检测；
- 光照检测；
- LED 补光；
- LCD1602 显示；
- 自动节能控制。

典型显示逻辑：

```text
未检测到人体 → LCD 显示 Waiting
检测到人体 → LCD 显示 Welcome
环境较暗且检测到人体 → 自动开启 LED 补光
按下门铃按键 → 蜂鸣器响铃
长时间无操作 → 系统自动进入节能状态
```

## 快速开始

### 1. 克隆仓库

```bash
git clone git@github.com:WU-ss/Facial_recognition_door_lock.git
```

### 2. 配置 ESP32-CAM 开发环境

在 Arduino IDE 中安装 ESP32 开发板支持，并选择合适的开发板，例如：

```text
AI Thinker ESP32-CAM
```

### 3. 修改 WiFi 信息

打开 `face_test.ino`，修改 WiFi 名称和密码：

```c
const char* ssid = "你的WiFi名称";
const char* password = "你的WiFi密码";
```

注意：ESP32-CAM 通常只支持 **2.4GHz WiFi**，不支持 5GHz WiFi。

### 4. 检查摄像头型号与引脚

在 `face_test.ino` 中确认：

```c
#define CAMERA_MODEL_AI_THINKER
```

并确保 `camera_pins.h` 中的引脚定义与你使用的 ESP32-CAM 模块一致。

### 5. 下载程序到 ESP32-CAM

使用 USB-TTL 模块连接 ESP32-CAM。下载程序时通常需要：

```text
GPIO0 接 GND
然后复位 / 上电进入下载模式
```

上传完成后，断开 GPIO0 与 GND，再重新复位。

### 6. 打开串口监视器

设置波特率：

```text
115200
```

复位 ESP32-CAM 后，串口监视器中会输出类似：

```text
WiFi连接成功
摄像头初始化完成，请使用 http://xxx.xxx.xxx.xxx 访问网站
```

### 7. 访问 Web 控制页面

在浏览器中输入串口打印的 ESP32-CAM IP 地址，进入本地 Web 控制页面。

常见操作：

1. 点击 `STREAM CAMERA` 开启视频流；
2. 输入用户名；
3. 点击 `ADD USER` 并将人脸对准摄像头，录入人脸；
4. 点击 `ACCESS CONTROL` 进入门禁识别模式；
5. 授权人脸识别成功后，继电器动作，电子锁打开。

### 8. 下载 STM32 辅助控制程序

使用 Keil 打开 STM32 工程，编译后下载到 STM32 控制板。连接红外传感器、光照传感器、LCD1602、蜂鸣器和 LED 后，即可实现辅助控制功能。

## 使用说明

1. 设备上电后，STM32 初始化 LCD、按键、LED、蜂鸣器和传感器。
2. ESP32-CAM 连接 WiFi，并启动本地 Web 服务器。
3. 电脑或手机连接同一 WiFi 后，通过浏览器访问 ESP32-CAM IP 地址。
4. 在 Web 页面录入授权人脸。
5. 进入门禁识别模式。
6. 当授权用户出现在摄像头前时，系统识别成功并自动开锁。
7. 非授权用户无法开锁。
8. 若红外检测到有人靠近，LCD 显示 `Welcome`；无人时显示 `Waiting`。
9. 若环境较暗且有人靠近，LED 自动补光。
10. 其他用户可按门铃按键触发蜂鸣器。

## 注意事项

1. ESP32-CAM 下载程序时需要进入下载模式，上传完成后要断开 GPIO0 与 GND 并复位。
2. 请确保 ESP32-CAM 与浏览器设备处于同一局域网。
3. WiFi 推荐使用 2.4GHz 网络。
4. 继电器和电控锁需要独立确认供电能力，避免因电流不足导致误动作。
5. 人脸识别效果会受光照、角度、距离和摄像头清晰度影响。
6. 本项目未实现活体检测，存在照片误识别风险，不建议直接用于真实安防。
7. 如果摄像头无法初始化，请检查摄像头排线、引脚配置和开发板型号。
8. 如果识别成功但门锁不动作，请检查 GPIO2、继电器触发电平、电源和锁体接线。
9. 如果 LCD 显示异常，请检查 STM32 接线、LCD1602 初始化和供电。
10. 如果补光灯不亮，请检查光照传感器输出、电路阈值和 LED 供电。
